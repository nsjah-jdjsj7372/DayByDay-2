# DayByDay — инструкция по публикации

Все файлы готовы для GitHub Pages. Это бесплатный хостинг от GitHub — после публикации вы получите постоянную ссылку вида:

```
https://ВАШ_ЛОГИН.github.io/daybyday/
```

По этой ссылке iPhone сможет установить приложение на экран Домой и оно будет открываться **без адресной строки браузера**, как полноценное приложение.

---

## Шаг 1. Создайте репозиторий

1. Зайдите на github.com → нажмите **New repository** (зелёная кнопка справа сверху).
2. Имя репозитория: `daybyday` (или любое другое — оно попадёт в ссылку).
3. Сделайте репозиторий **Public** (обязательно для бесплатного GitHub Pages).
4. Не отмечайте "Add README" — оставьте пустым.
5. Нажмите **Create repository**.

## Шаг 2. Загрузите файлы

На странице нового репозитория:

1. Нажмите **uploading an existing file** (ссылка прямо на странице) или **Add file → Upload files**.
2. Перетащите туда **все файлы и папки** из этого комплекта:
   - `index.html`
   - `manifest.json`
   - `sw.js`
   - папку `icons` (со всеми файлами внутри)
3. Внизу нажмите **Commit changes**.

> Важно: структура папок должна сохраниться — `icons/` должна остаться папкой, а не превратиться в отдельные файлы в корне.

## Шаг 3. Включите GitHub Pages

1. В репозитории откройте **Settings** (вкладка сверху).
2. В левом меню найдите **Pages**.
3. В разделе **Build and deployment → Source** выберите **Deploy from a branch**.
4. В **Branch** выберите `main` (или `master`) и папку `/ (root)`.
5. Нажмите **Save**.
6. Подождите 1–2 минуты — GitHub соберёт сайт.
7. Обновите страницу Settings → Pages — сверху появится зелёная плашка с вашей ссылкой:
   ```
   Your site is live at https://ВАШ_ЛОГИН.github.io/daybyday/
   ```

## Шаг 4. Установите на iPhone

1. Откройте полученную ссылку **в Safari** на iPhone (важно: именно в Safari, не в Chrome — на iOS только Safari умеет устанавливать PWA).
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой вверх) в нижней панели.
3. В списке найдите **«На экран “Домой”»** (On Home Screen) — возможно, нужно прокрутить список вниз.
4. Подтвердите имя **DayByDay** → нажмите **«Добавить»**.
5. На экране Домой появится иконка DayByDay. При запуске оно откроется в полноэкранном режиме — без адресной строки и кнопок Safari, как настоящее приложение.

---

## Важно знать про данные

Приложение хранит все данные (планы, цели, заметки) **локально в памяти браузера на телефоне** (localStorage), а не в облаке. Это значит:

- Данные не передаются никуда и видны только вам — это плюс для приватности.
- Если вы удалите приложение с экрана Домой или очистите данные Safari, записи **пропадут безвозвратно**. Резервных копий в облаке нет.
- Данные не синхронизируются между разными устройствами (например, между iPhone и iPad) — это будет два независимых хранилища.

Если в будущем понадобится синхронизация между устройствами или облачное резервное копирование — это отдельная доработка (потребуется база данных), дайте знать, помогу это добавить.

## Обновление приложения в будущем

Если потребуется что-то поправить в приложении:
1. Внесите изменения в `index.html`.
2. Загрузите обновлённый файл в тот же репозиторий на GitHub (тот же путь — он перезапишет старый).
3. GitHub Pages обновится автоматически за 1-2 минуты.
4. На телефоне может потребоваться закрыть приложение полностью (свайпом) и открыть заново — Service Worker иногда кэширует старую версию на пару дней.
